# medagent 服务 · 后端接入指南

本文面向**后端开发者**，说明如何把本仓库的无人医院 AI 诊疗服务接入你们的系统。读完你应能：起服务、按就诊时序驱动 HTTP 接口、拿到结构化诊疗结果、并把会话纪要导出持久化。

- **唯一公开包**：`agent`（导入路径 `medagent/agent`，Go module 仍为 `medagent`），以 HTTP/JSON 对外；底层决策层固定编排在 `internal/`，外部不可见、不可直接调用。
- Go 1.22，零外部依赖（仅标准库）。
- 内存持有多轮会话（按 `session_id` 路由），**持久化由后端负责**（会话结束后用 Export 导出 `SessionRecord` 存库）。

## 1. 起服务

### 1.1 独立进程（推荐）

```bash
# provider=deepseek，API key 由环境变量提供
DEEPSEEK_API_KEY=<your_key> go run ./cmd/server -provider deepseek

# 完整 flags（均有默认值）
go run ./cmd/server \
  -addr     :8080            \   # 监听地址，默认 :8080
  -provider deepseek         \   # deepseek | qwen | openai
  -model    deepseek-chat    \   # 模型名
  -base-url ""               \   # 设此项则忽略 provider，直接用该 URL
  -log-dir  ./logs               # 诊疗日志目录；设空字符串则不写日志
```

provider 与对应 env 变量：

| provider | env 变量 |
|---|---|
| `deepseek` | `DEEPSEEK_API_KEY` |
| `qwen` | `DASHSCOPE_API_KEY` |
| `openai` | `OPENAI_API_KEY` |

### 1.2 嵌入 Go 程序（库用法）

```go
import "medagent/agent"

svc, err := agent.New(agent.Config{
    Provider: "deepseek",
    APIKey:   os.Getenv("DEEPSEEK_API_KEY"),
    Model:    "deepseek-chat",
    LogDir:   "./logs",                     // 可选；空字符串则不写日志
})
if err != nil {
    log.Fatal(err)
}
defer svc.Close()

// svc.Handler() 返回 http.Handler，自挂载到你的 mux
http.Handle("/ai/", http.StripPrefix("/ai", svc.Handler()))
http.ListenAndServe(":8080", nil)
```

## 2. HTTP 端点总表

所有响应 `Content-Type: application/json`。

| 方法 | 路径 | 作用 | 成功状态码 |
|---|---|---|---|
| `POST` | `/sessions` | 开始新会话 | `200` |
| `POST` | `/sessions/{id}/patient-say` | 患者发言 → AI 下一步 | `200` |
| `POST` | `/sessions/{id}/test-results` | 回填检验结果 → AI 下一步 | `200` |
| `POST` | `/sessions/{id}/drug-info` | 回填药品规格 → AI 下一步 | `200` |
| `POST` | `/sessions/{id}/purchase-result` | 回报购药结果 → AI 下一步 | `200` |
| `POST` | `/sessions/{id}/vitals` | 上报体征 → 守护判断 | `200` |
| `GET` | `/sessions/{id}/record` | 导出会话纪要 | `200` |
| `DELETE` | `/sessions/{id}` | 销毁会话（立即生效） | `204` |

### 错误状态码

| HTTP 状态码 | 含义 | 对应错误 |
|---|---|---|
| `400` | 请求体 JSON 无效 | 解析失败 |
| `404` | 会话不存在 | `ErrSessionNotFound` |
| `409` | 会话已结束 / 当前步骤不匹配 | `ErrSessionClosed` / `ErrWrongStep` |
| `504` | 上游 LLM 超时或请求被取消 | `context.DeadlineExceeded` / `context.Canceled` |
| `502` | 其余上游错误（LLM 故障、模型输出非法） | `ErrUpstream` / `ErrModelOutput` |

错误响应体：`{"error": "<message>"}`

## 3. 端点详情

### 3.1 POST /sessions — 开始会话

**请求体：**
```json
{
  "profile": { "age": 35, "gender": "male", "allergies": [] },
  "initial": true,
  "prior":   []
}
```

| 字段 | 类型 | 说明 |
|---|---|---|
| `profile` | `object`（任意 JSON 对象） | 患者基本资料，透传给 AI |
| `initial` | `bool` | `true` = 初诊；`false` = 复诊 |
| `prior` | `array<SessionRecord>` | 复诊时回传之前就诊的 `Export` 导出纪要（初诊传 `[]`） |

**响应：**
```json
{ "session_id": "20260624-153012-a1b2c3d4" }
```

### 3.2 POST /sessions/{id}/patient-say — 患者发言

**请求体：**
```json
{ "message": "我发烧两天了，体温 38.5°" }
```

**响应：** `Step` 对象（见 §4）

仅在会话处于问诊阶段时合法；否则返回 `409`。

### 3.3 POST /sessions/{id}/test-results — 回填检验结果

**请求体：**
```json
{
  "results": [
    { "item": "血常规-白细胞", "value": "11.2×10⁹/L" },
    { "item": "CRP",           "value": "28 mg/L" }
  ]
}
```

仅在收到 `kind=NEED_TESTS` 后调用；否则返回 `409`。

**响应：** `Step` 对象

### 3.4 POST /sessions/{id}/drug-info — 回填药品规格

**请求体：**
```json
{
  "infos": [
    { "name": "布洛芬缓释胶囊", "spec": "每盒24粒×0.3g" }
  ]
}
```

仅在收到 `kind=DRUG_QUERY` 后调用；否则返回 `409`。

**响应：** `Step` 对象（通常为 `PURCHASE`）

### 3.5 POST /sessions/{id}/purchase-result — 回报购药结果

**请求体：**
```json
{
  "results": [
    { "name": "布洛芬缓释胶囊", "bought": true,  "quantity": 2 },
    { "name": "阿莫西林胶囊",   "bought": false, "quantity": 0 }
  ]
}
```

仅在收到 `kind=PURCHASE` 后调用；否则返回 `409`。

**响应：** `Step` 对象（通常为 `DONE`）

### 3.6 POST /sessions/{id}/vitals — 上报体征

**请求体：**
```json
{ "vitals": { "heart_rate": 128, "spo2": 91, "bp": "160/100" } }
```

任意阶段均可调用（已结束的会话返回 `409`）。响应为 `Step{kind: "OK"}` 或 `Step{kind: "EMERGENCY"}`。

### 3.7 GET /sessions/{id}/record — 导出会话纪要

响应：`SessionRecord` 对象（见 §6）。会话进行中也可调用（实时快照）。

### 3.8 DELETE /sessions/{id} — 销毁会话

立即从内存删除，返回 `204 No Content`（无响应体）。销毁前建议先 Export。

## 4. Step：AI 下一步指令

患者发言、回填检验、购药结果等接口均返回 `Step` 对象。以下为 `kind=ASK` 时的实际响应示例：

```json
{
  "kind":       "ASK",
  "doctor_say": "您发烧几天了？有无其他症状？"
}
```

| 字段 | JSON key | 类型 | 说明 |
|---|---|---|---|
| `Kind` | `kind` | `string` | 步骤类型（见下表） |
| `DoctorSay` | `doctor_say` | `string` | 医生话术（`ASK` 时非空） |
| `TestItems` | `test_items` | `[]string` | 检验项目列表（`NEED_TESTS` 时非空）；**当前系统固定为 `["血常规"]`**——后端只需对血常规回填结果，无需动态适配其他检验项 |
| `DrugNames` | `drug_names` | `[]string` | 待查规格的药名（`DRUG_QUERY` 时非空） |
| `Orders` | `orders` | `[]DrugOrder` | 购药清单（`PURCHASE` 时非空） |
| `Emergency` | `emergency` | `string` | 急症原因（`EMERGENCY` 时非空） |
| `Result` | `result` | `Result` 对象 | 诊疗结果（`DONE` 时非空） |

### StepKind 取值

| `kind` | 含义 | 后端动作 |
|---|---|---|
| `ASK` | AI 继续问诊 | 把 `doctor_say` 展示给患者，等患者下一句 → `/patient-say` |
| `NEED_TESTS` | 需要检验 | 驱动检验子系统执行 `test_items`，拿到结果后 → `/test-results` |
| `DRUG_QUERY` | 需查询药品规格 | 用 `drug_names` 逐一查库获取每盒规格，回填 → `/drug-info` |
| `PURCHASE` | 需要购药 | 驱动药房/支付子系统处理 `orders`，结果后 → `/purchase-result` |
| `EMERGENCY` | 急症打断 | `emergency` 字段含原因，立即告知患者转急诊；会话自动关闭 |
| `DONE` | 就诊完成 | `result` 含完整诊疗结果；调 `/record` 导出纪要后 `/DELETE` 销毁 |
| `OK` | 体征正常 | 仅 `/vitals` 返回，守护未触发，继续当前流程 |

omitempty 字段仅在对应 kind 时出现，其余字段在响应 JSON 中不存在。

### DrugOrder

```json
{ "name": "布洛芬缓释胶囊", "quantity": 2 }
```

`quantity` 为**购买盒数**（`DrugOrder` 与 `Medication` 中含义相同）。

### Result（DONE 时的诊疗结果）

```json
{
  "final":    "ADVICE",
  "diagnosis": { "name": "急性上呼吸道感染", "basis": "发热+咽痛+血象", "confidence": 0.88 },
  "plan":     "MEDICATION",
  "medications": [
    { "name": "布洛芬缓释胶囊", "dosage": "0.3g", "schedule": "每日两次", "quantity": 2 }
  ],
  "advice":   "多休息，多饮水，三天后如不好转来复诊"
}
```

| 字段 | JSON key | 说明 |
|---|---|---|
| `Final` | `final` | 就诊终态，仅两值：`ADVICE`（给出医嘱/用药即结束）/ `REFERRAL`（需转诊） |
| `Diagnosis` | `diagnosis` | 诊断（含 `name`/`basis`/`confidence` 0~1） |
| `Plan` | `plan` | 处置类型：`MEDICATION` / `ADVICE_ONLY` / `REFERRAL`（注意与 `final` 不同；院内治疗执行不闭环，无 TREATMENT） |
| `Medications` | `medications` | 处方药品列表（`MEDICATION` 时非空） |
| `Advice` | `advice` | 医嘱文本，恒非空 |

## 5. 一次就诊时序

以下时序图省略 `/sessions/{id}` 前缀（除 `/sessions` 外的所有路径）；详见端点总表。

```
POST /sessions           → {session_id}
    │
    ▼ 问诊阶段（可多轮）
POST /patient-say        → {kind: "ASK", doctor_say: "..."}   （循环直到非 ASK）
    │
    ├── kind = NEED_TESTS
    │       │
    │       ▼ 后端执行检验
    │   POST /test-results   → 下一步（可再次 ASK 或进入 PURCHASE/DONE）
    │
    ├── kind = DRUG_QUERY
    │       │
    │       ▼ 后端查药品规格
    │   POST /drug-info       → 下一步（通常 PURCHASE）
    │
    ├── kind = PURCHASE
    │       │
    │       ▼ 后端驱动药房/支付
    │   POST /purchase-result → {kind: "DONE", result: {...}}
    │
    ├── kind = DONE          → 直接结束（无需购药）
    │
    └── kind = EMERGENCY     → 急症打断，立即转急诊（任意阶段均可出现）

    （任意阶段）
POST /vitals             → {kind: "OK"} 或 {kind: "EMERGENCY"}

    就诊完成后：
GET  /record             → SessionRecord（含完整纪要）
DELETE /sessions/{id}   → 204
```

**典型多轮问诊示例：**

```
POST /patient-say {"message":"我头疼"}      → ASK: "头疼多久了？"
POST /patient-say {"message":"三天"}        → ASK: "有无发烧？"
POST /patient-say {"message":"有，38.5度"}  → NEED_TESTS: ["血常规"]
POST /test-results {"results":[...]}        → DRUG_QUERY: ["布洛芬缓释胶囊"]
POST /drug-info {"infos":[{"name":"布洛芬缓释胶囊","spec":"每盒24粒×0.3g"}]}
                                            → PURCHASE: [{name:"布洛芬缓释胶囊",quantity:2}]
POST /purchase-result {"results":[...]}     → DONE: {result:{...}}
```

## 6. 会话纪要（SessionRecord）

`GET /sessions/{id}/record` 返回：

```json
{
  "session_id": "20260624-153012-a1b2c3d4",
  "initial":    true,
  "started_at": "2026-06-24T15:30:12Z",
  "ended_at":   "2026-06-24T15:42:08Z",
  "profile":    { "age": 35, "gender": "male" },
  "turns": [
    { "at": "2026-06-24T15:30:15Z", "kind": "patient",          "text": "我头疼" },
    { "at": "2026-06-24T15:30:17Z", "kind": "doctor",           "text": "头疼多久了？" },
    { "at": "2026-06-24T15:31:00Z", "kind": "test_request",     "text": "血常规" },
    { "at": "2026-06-24T15:38:00Z", "kind": "test_result",      "text": "血常规-白细胞: 11.2×10⁹/L" },
    { "at": "2026-06-24T15:40:00Z", "kind": "purchase_request", "text": "[{布洛芬缓释胶囊 2}]" },
    { "at": "2026-06-24T15:41:00Z", "kind": "purchase_result",  "text": "布洛芬缓释胶囊 购买=true 数量=2" },
    { "at": "2026-06-24T15:42:08Z", "kind": "advice",           "text": "多休息，多饮水" }
  ],
  "outcome": { "final": "ADVICE", "plan": "MEDICATION", ... }
}
```

| 字段 | JSON key | 说明 |
|---|---|---|
| `SessionID` | `session_id` | 会话唯一 ID |
| `Initial` | `initial` | 是否初诊 |
| `StartedAt` | `started_at` | 开始时间（秒精度，RFC3339） |
| `EndedAt` | `ended_at` | 结束时间（omitempty；进行中为 null） |
| `Profile` | `profile` | 患者资料原始 JSON |
| `Turns` | `turns` | 对话回合列表，`at` 为秒精度时间戳 |
| `Outcome` | `outcome` | 最终 `Result`（omitempty；进行中或急症终止时为 null） |

`RecordedTurn.Kind` 取值：`patient` / `doctor` / `diagnosis` / `test_request` / `test_result` / `drug_query` / `drug_info` / `purchase_request` / `purchase_result` / `advice` / `warn` / `emergency`。其中 `warn` 表示一条需后端复核的告警（如某购药盒数为 0，规格缺失或换药所致）。

**复诊接入：** 复诊时把上次（及更早）就诊的 `SessionRecord` 以数组形式回传到 `POST /sessions` 的 `prior` 字段。服务将历次摘要渲染后喂给 AI，使其了解病史。

## 7. Config 字段

```go
agent.Config{
    Provider:        "deepseek",       // deepseek | qwen | openai（设 BaseURL 时忽略）
    APIKey:          "sk-...",         // 必填
    Model:           "deepseek-chat",  // 必填
    BaseURL:         "",               // 自定义端点（任意 OpenAI 兼容 API）；非空时忽略 Provider
    LogDir:          "./logs",         // 诊疗日志目录；空字符串=不写日志
    Timeout:         30 * time.Second, // 单次 LLM 调用超时；0=默认 60s
    SessionTTL:      30 * time.Minute, // 无活动超时；0=默认 30min，到期自动回收
    DisableGuardian: false,            // true=关闭急症守护（默认开启）
}
```

`APIKey` 与 `Model` 必填；`Provider` 与 `BaseURL` 二选一（`BaseURL` 优先）；其余可选。

## 8. 错误语义

```go
import "errors"
import "medagent/agent"

step, err := svc.PatientSay(ctx, id, "...")
switch {
case err == nil:
    // 用 step
case errors.Is(err, agent.ErrSessionNotFound):
    // 会话不存在 → HTTP 会映射 404
case errors.Is(err, agent.ErrSessionClosed):
    // 会话已结束 → 409
case errors.Is(err, agent.ErrWrongStep):
    // 当前步骤不允许此调用 → 409
case errors.Is(err, context.DeadlineExceeded), errors.Is(err, context.Canceled):
    // LLM 超时或请求取消 → 504
case errors.Is(err, agent.ErrUpstream):
    // LLM 传输/非 2xx/响应格式错误 → 502
case errors.Is(err, agent.ErrModelOutput):
    // 模型输出重试后仍非法 → 502
}
```

HTTP 服务已自动把上述 sentinel 映射为对应状态码；Go 库用法时按 sentinel 自行处理。

## 9. 日志（可选）

设置 `Config.LogDir`（或 `-log-dir` flag）后，服务按 `session_id` 为每次就诊写 JSONL 日志，每条 LLM 调用一行：

```
./logs/20260624-153012-a1b2c3d4.jsonl
```

- 日志失败**绝不影响诊疗**（错误仅写 stderr）。
- 日志含完整医患对话，属医疗数据——落点权限与脱敏由后端决定；建议加入 `.gitignore`。
- `LogDir` 目录若不存在，`New()` 会自动创建；目录创建失败则 `New()` 返回错误。

## 10. 急症守护

- 急症守护**默认开启**（`DisableGuardian: false`）。
- 任何 `/patient-say`、`/test-results`、`/drug-info`、`/purchase-result` 调用都与守护并发运行；守护命中时立即中止主流程，返回 `Step{kind: "EMERGENCY", emergency: "<原因>"}`，会话自动关闭。
- `/vitals` 用于主动推送体征（如监护仪数据），返回 `Step{kind: "OK"}` 或 `Step{kind: "EMERGENCY"}`。
- 设 `DisableGuardian: true` 后：所有接口均不触发急症判断，`/vitals` 永远返回 `OK`。
- 守护错误 fail-open：守护自身 LLM 调用出错时不打断主流程，诊疗照常继续。

## 11. 接入边界（后端职责）

| 职责 | 说明 |
|---|---|
| **初诊/复诊判定** | 由后端决定 `initial` 字段，并把历史 `SessionRecord` 数组放入 `prior` |
| **检验子系统** | 收到 `NEED_TESTS` 后后端驱动检验，完成后调 `/test-results` 回填 |
| **药品/支付子系统** | 收到 `PURCHASE` 后后端驱动药房/支付，完成后调 `/purchase-result` 回报 |
| **持久化** | 会话结束后调 `/record` 导出 `SessionRecord` 存库；服务重启后内存会话丢失 |
| **鉴权网关** | HTTP 服务无内置鉴权，需在前置网关/反向代理处处理 |
| **并发与熔断** | 服务内部已有轮数护栏（问诊≤20、收敛≤10、处置≤6，处置含查规格/购药/终决轮）；业务层超时/熔断由外部处理 |
| **会话 TTL** | 内存会话无活动超过 `SessionTTL`（默认 30min）后自动回收；正式下线前请调 `/DELETE` |
